# ЧАСТЬ A. АДМИН-ПАНЕЛЬ (9 вкладок)

Доступ: только `role === 'admin'`. Загрузка: `GET /admin/{dashboard,bookings,cars,users,settings}`, `GET /faq`, `GET /blog?all=1`.
Просмотр документа `viewDoc(url)`: base64 → Blob → новая вкладка (или `window.open`).

## A1. Дашборд `GET /admin/dashboard` ⬜
- ⬜ 6 KPI-карточек: выручка всего, выручка за месяц, всего броней, клиентов, авто (published/total), ожидают подтверждения
- ⬜ График «Выручка по месяцам» (6 мес, `revenue_by_month[]`)
- ⬜ Топ-5 авто по выручке (`top_cars[]`: фото, аренд, выручка)
- ⬜ Календарь броней на 14 дней (`calendar[]`: строки-авто × 14 дней, подсветка по статусу)
- ⬜ Quick stats: средний чек, завершено, активных, отменено

## A2. Брони `GET/POST/PATCH /admin/bookings` 🟡 (список ✅, остальное ⬜)
- ✅ Список-таблица (авто, клиент, даты, сумма, статус)
- ⬜ Поиск (#ID, авто, имя/email/тел клиента, менеджер)
- ⬜ Фильтр статуса: Все/Ожидают/В аренде/Завершённые/Отменённые
- ⬜ Инлайн-смена этапа воронки в строке (`PATCH {stage}`)
- ⬜ Кнопки по статусу: Выдать (→active), Завершить (→completed), Отменить (→cancelled)
- ⬜ Клик по клиенту → карточка клиента; по авто → карточка авто
- ⬜ Бейдж «вручную» при `source==='manual'`
- ⬜ **Модалка карточки брони** (редактирование): сумма, этап, даты начала/конца (datetime-local, **с фиксом UTC→local `isoToLocalInput`, −3ч МСК**), менеджер, адрес подачи, примечание → `PATCH /admin/bookings/:id`
- ⬜ **Модалка ручного создания брони**: авто, клиент (существующий/новый ФИО+тел+email), даты, сумма, этап, менеджер, адрес подачи, чекбокс «нужна доставка», примечание → `POST /admin/bookings`
- ⬜ Воронка STAGES (7): new, docs, prepay, manager, issued, completed, cancelled

## A3. Автомобили `GET/POST/PATCH/DELETE /admin/cars` (+reorder, +photos) ⬜

- ⬜ Статусы: published/hidden/pending/rejected
- ⬜ Порядок ▲/▼ (`POST /admin/cars/reorder`)

# ЧАСТЬ B. ЛК КЛИЕНТА (8 вкладок)

Старое: `Account.jsx` (8 вкладок), `AccountChat.jsx`, `ChatBox.jsx`. Вкладки:
`overview, bookings, history, favorites, bonuses, documents, profile, chat`.
Открытие по хешу URL (`/account#documents`). Все запросы через `/api`, cookie-сессия.
На Metronic → раздел **Store-Client** (роль `user`).

## B0. Каркас ЛК ⬜
- ⬜ Сайдбар: «Забронировать авто» (/catalog), «На сайт» (/), блок юзера (аватар+имя+уровень клуба), навигация с бейджами (активные брони, избранное, непрочитанные), «Выйти»
- ⬜ **Аватар** (на любой вкладке): загрузка ≤1 МБ base64 → `PATCH /me {avatar_url}`; плейсхолдер ui-avatars
- ⬜ **Баннер верификации** сверху: «нужно подать документы»/«документы на проверке» (по `is_verified` + наличию доков)
- ⬜ Поллинг непрочитанных 15с (`GET /chat/unread-count`)

## B1. Обзор (overview) `GET /me/bookings,/me/favorites,/me/points` ⬜
- ⬜ 3 карточки: Аренд всего, Активных (active/pending), Потрачено (сумма completed, млн ₽)
- ⬜ + активные брони + история (как на своих вкладках)

## B2. Бронирования (bookings) ⬜
- ⬜ 3 карточки статистики + список активных (active/pending): фото/название/год/кузов, даты, сумма, статус
- ⬜ «Детали авто» (/car/:id), **«Отменить бронь»** (confirm → `PATCH /bookings/:id {status:cancelled}`)
- ⬜ (продления в ЛК НЕТ — только отмена)

## B3. История (history) ⬜
- ⬜ Таблица завершённых/отменённых: авто, период, стоимость, статус-тег

## B4. Избранное (favorites) ⬜
- ⬜ Сетка карточек авто из `GET /me/favorites` (id) + данные из `/cars`
- ⬜ Добавление/удаление — на карточке авто (сердечко): `POST/DELETE /me/favorites/:carId` (оптимистично)

## B5. Бонусы (bonuses) `GET /me/points` ⬜
- ⬜ Баланс баллов (1 балл = 1 ₽) + описание программы (приветственный 10%, кэшбэк 5%, списание до 100%)
- ⬜ История начислений (дата, назначение, сумма +/−)

## B6. Документы (documents) — верификация ⬜
- ⬜ 4 документа (passport_url, passport_page_url, registration_url, license_url): загрузка ≤2 МБ base64 (`image/*,.pdf`) → `PATCH /me {[field]}`; статус загружено/нет; просмотр (viewDoc)
- ⬜ Если `is_verified` — плашка «пройдена», документы заблокированы (только просмотр); иначе — «загрузите/можно менять»

## B7. Профиль (profile) ⬜
- ⬜ Редактирование: имя, телефон, email, дата рождения → `PATCH /me`
- ⬜ Read-only: уровень клуба (по role+created_at), верификация СБ
- ⬜ (смены пароля и подтверждения email в ЛК НЕТ — они в auth-флоу вне кабинета)

## B8. Чат с менеджером (chat) — мульти-треды ⬜
- ⬜ Треды: общий «Поддержка» + по машинам; табы с бейджами (`GET /chat/threads`, `POST /chat/threads {car_id?}`)
- ⬜ Лента (ChatBox): SSE `/chat/stream` + polling 12с, отправка (`POST .../messages {body,attachment}`), отметка read
- ⬜ Вложения (image/pdf ≤2 МБ base64, превью), блок ввода если тред закрыт

## B (вне вкладок ЛК, но связано)
- ⬜ Отзывы на авто — на странице `/car/:id` (`GET/POST /cars/:id/reviews {rating,text}`), не в ЛК
- ⬜ Клубная карта — страница `/club` (уровни по сумме трат), не вкладка ЛК
- ⬜ Auth-флоу: register/login/login-verify/verify-code/resend-code/logout (`POST /auth/*`) — страницы входа/регистрации

## Общие технические заметки для переноса
- Загрузка файлов везде — **base64 (data-URL), лимит 2 МБ** (фото авто, обложка блога, вложения чата, документы клиента в ЛК). Кандидат на реальный upload-эндпоинт при переносе.
- `isoToLocalInput` — фикс сдвига дат −3ч (МСК). НЕ терять при переносе datetime-полей.
- Роль и `is_verified` меняются тем же `PATCH /admin/users/:id`.
- Баллы через `prompt()` → на Metronic заменить нормальным модальным вводом.
